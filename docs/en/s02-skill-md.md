# s02 · SKILL.md parser & renderer

> What this teaches: a round-trippable YAML-frontmatter + Markdown parser for `SKILL.md`, the single file an agent reads to learn that a CLI-Anything harness exists, what it does, and which words should trigger it.

## Problem

Every harness in CLI-Anything ships a `SKILL.md`. Upstream `cli-anything-plugin/skill_generator.py` writes them; downstream agents (Claude Code, Pi, Codex) read them. The format is deceptively simple — three-dash fenced YAML frontmatter, then opaque markdown — but the bytes matter. The generator emits folded-scalar `>-` blocks to keep `description` lines short, and a parser that re-emits via `yaml.Marshal` will silently rewrite them to plain strings, breaking any reproducible-build CI that diffs the generated file against the previous commit. So our parser has two jobs: extract the structured fields, AND preserve the literal frontmatter bytes so `Parse → Render` is identity.

## Solution

Two types and two functions. `SkillMeta` (the structured view of the frontmatter) is a plain Go struct with `yaml` tags. `Skill` wraps it with the body bytes AND an unexported `raw []byte` that holds the verbatim frontmatter slice. `Parse(data []byte) (Skill, error)` locates the two `---\n` fences, snapshots the bytes between them into `raw`, then `yaml.Unmarshal`s a *copy* into `SkillMeta`. `Render(s Skill) []byte` emits `---\n` + (`s.raw` if present else `yaml.Marshal(s.Meta)`) + `---\n` + body. The raw-bytes fast path is what gives us the round-trip guarantee; the marshal path covers freshly constructed skills (s03 will produce these). Validation is intentionally minimal — only `name` is required, mirroring upstream's `_canonical_skill_name` invariant.

## How It Works

```text
bytes ─▶ Parse ─┬─▶ find first  '---\n'   (offset 0; else errMissingDelim)
                ├─▶ find second '---\n'   (else errMissingDelim)
                ├─▶ snapshot raw = bytes between fences
                ├─▶ yaml.Unmarshal(raw) → SkillMeta
                ├─▶ require Meta.Name != "" (else errMissingName)
                └─▶ Skill{Meta, Body, raw}

Skill ─▶ Render ─┬─▶ "---\n"
                 ├─▶ raw (if len>0) else yaml.Marshal(Meta)
                 ├─▶ "---\n"
                 └─▶ Body
```

The interesting 30 lines of `agents/s02-skill-md/skill.go`:

```go
func Parse(data []byte) (Skill, error) {
    if !bytes.HasPrefix(data, delim) { return Skill{}, errMissingDelim }
    rest := data[len(delim):]
    end  := bytes.Index(rest, delim)
    if end < 0 { return Skill{}, errMissingDelim }
    frontRaw := rest[:end]
    body     := rest[end+len(delim):]
    var meta SkillMeta
    if err := yaml.Unmarshal(frontRaw, &meta); err != nil {
        return Skill{}, fmt.Errorf("skill: yaml: %w", err)
    }
    if meta.Name == "" { return Skill{}, errMissingName }
    return Skill{Meta: meta, Body: body, raw: frontRaw}, nil
}
```

Three non-obvious points:

1. **`raw` is the round-trip secret.** Without it, the third test (folded-scalar preservation) fails — yaml.v3 will normalize `name: >-\n  cli-anything-anygen\n` to `name: cli-anything-anygen\n`. The bytes diverge, downstream diffs explode, CI fires. Keep the originals; only re-emit when the caller built a `Skill` from scratch.
2. **Typed errors, not just messages.** `errMissingDelim` and `errMissingName` are sentinels so callers (s03's generator, s08's validator) can `errors.Is` against them. The test suite uses this to assert on the two distinct failure modes that the upstream's `verify-plugin.sh` cares about.
3. **Body is `[]byte`, not `string`.** SKILL.md files can contain non-UTF8 fragments inside code blocks (binary protocol examples, hex dumps in `anygen` docs). Converting to `string` would be lossless today but locks us out of future binary-aware uses; keeping `[]byte` costs nothing.

## What Changed (vs. s01)

s01 gave us `Dispatch` and the JSON envelope. s02 reuses both (re-declared, because each session is a self-contained Go module — see `cli.go`) and adds a new pure module: `skill.go`. The CLI surface gains `parse` and `render` subcommands; neither needs anything from s01 it didn't already have. The only new dependency is `gopkg.in/yaml.v3`. Net adds: ~120 LOC of Go, 5 tests, one Markdown sample under `/tmp` for the demo target.
